r3553 | vasiljevic | 2014-02-18 19:43:29 +0100 (Tue, 18 Feb 2014) | 9 lines

Graph editor: improvements related to box content editing

When you are editing box content and you have made some changes:
- if you click on the graph background, changes will be stored in the appropriate box(es) and all boxes will be deselected
- if you click on a box, changes will be  stored in the appropriate box(es) and the clicked box will be selected without altering transitions; so, you can just click on the next box you want to edit (if you don't make any changes in selected box content, the Graph editor will behave as usual: it will draw/remove transitions)
- if you do "Save" or "Save as", changes will be stored in the appropriate box(es) before saving
- in the all previous cases if there is an error in the box content, the operation will be canceled and you go back to the edit field, to fix the error
- if you want to cancel editing, just hit ESC
